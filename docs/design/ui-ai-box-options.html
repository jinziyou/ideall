中央 AI 框 · 两种方案

常驻顶部框 ＆ 浮层 ⌘K — 选一个

同样在 D 皮肤 + A 布局 的 home 概览语境下，把「中央 AI / 命令入口」做成两种形态给你挑。其余（图标轨、上下文栏、便当全保留）完全一致，只对比这一个元素。默认主题=跟随系统 / 回退亮色。

一句话区别：方案 1 在 main 顶部常驻一个 AI 框（看得见、邀请输入）；方案 2 不占 main 空间，用 ⌘K 浮层把同一能力做成 Raycast 式命令台（内容区更大、更干净）。

方案 1 常驻顶部中央 AI 框

main 顶部固定一个 AI / 搜索 / 命令框（ima 式）。始终可见，鼓励提问与搜索，对新手发现性最好；代价是占用 main 顶部约一行高度。

方案 2 纯浮层 ⌘K 命令台

main 顶部不放框，按 ⌘K（或点轨上的触发器）唤起居中浮层命令台——搜索、跳转、执行、回流、提问全在里面（Raycast / Spotlight 式）。内容区最大、最干净，最贴「命令台是脊柱」与极客受众。

我的推荐：方案 3 = 混合（两者长处）

底层用方案 2 的浮层 ⌘K 作引擎（跳转/执行/回流/提问统一、内容区最大、最贴极客与「脊柱」原则）；顶部不放完整框，而是保留一个纤细的「⌕ 搜索 / 命令 ⌘K」触发 pill（即方案 2 图中页头右上那个）补足发现性。既干净专业，又不丢入口。

若你更看重「新手一眼会用 / 中文中央框习惯」→ 选方案 1；更看重「干净 / 键盘 / 可扩展」→ 选方案 2；想兼得 → 方案 3 混合（推荐）。
