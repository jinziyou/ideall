D 皮肤打底 + A 页面布局

把 A · 墨阶指挥所 的高效骨架（图标轨 + 上下文栏 + ⌘K 命令台脊柱）和 D · 便当工作台 的亲和皮肤（亮色、软圆角便当卡、中央 AI 框、三辐柔和色）合到一起：专业效率的结构 + 现代柔和的质感。下面把这套系统贯穿到全应用每个页面，请你确认观感后我再落地成代码。

布局 ｜ A：图标轨 + 上下文栏 + 命令台 皮肤 ｜ D：亮色软圆角便当 + 中央 AI 框 范围 ｜ 整个应用全部页面 平台 ｜ 桌面 + 移动

A 取「布局 / 结构」

  • 竖向图标轨（我的/资讯/社区/工具/AI + 本机/头像）替代横向 header
  • 上下文栏随轨切换：显示该区子导航、计数、筛选
  • ⌘K 命令台为脊柱：跳转 + 执行 + 订阅 + 回流统一入口
  • home=「指挥台」信息密度：统计 + 实时回流流 + 钉住工具

D 取「皮肤 / 质感」

  • 亮色底 + 白色软圆角便当卡（18px squircle）+ 柔阴影
  • main 顶部常驻中央 AI / 搜索 / 命令框（✦ 提问入口）
  • 便当看板：磁贴大小 = 重要度，可增删重排
  • 三辐色作柔和点缀（资讯蓝 / 社区绿 / 工具紫）+ 友好靛蓝强调

只换 UI 层，复用现有数据层与契约（@protocol / IndexedDB / ServerPort）。在 dev 分支推进，保留暗色变体与既有约定（默认 Server Component、文案中文、protocol 纯度）。

皮肤 token（亮/暗） app/globals.css 的 @theme：改 --background/--card/--radius/--accent + 三辐色；保留 .dark
UI 原语 / 共享头 components/ui（shadcn 复用）+ components/shared（app-header / service-header 调整）

建议落地顺序

① 设计 token 与全局壳层（rail + 上下文栏 + 命令台 + 移动底栏）→ ② home 概览便当 → ③ home 子页（订阅 / 书签 / 资源 / 发布 / AI）→ ④ 三辐（资讯 / 社区 / 工具）→ ⑤ 暗色变体打磨 + 移动联调。每步可独立提交，随时预览。
