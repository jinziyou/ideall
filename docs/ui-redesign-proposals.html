页面布局与 UI 重构 · 方案集

把「中枢辐射」做成一眼认得出的产品

这份文档先拆解 10 类主流同类产品的布局与 UI，提炼可迁移到 ideall 的原则，再给出 4 套差异明显的全新视觉 + 布局方向（含桌面与移动示意图）。每套都覆盖全局壳层、home 中枢与 info / community / tool 三辐，供你横向对比后挑一套，我再落地成 React/Tailwind 代码。

范围 ｜ 整个应用全部页面 形式 ｜ 设计说明 + 示意图 视觉 ｜ 大胆探索全新视觉 平台 ｜ 桌面 + 移动都细做 方案数 ｜ 4 套
当前形态回顾。 顶部 h-16 固定 header（logo + 横向导航 + ⌘K/主题/账户）；/home 为左侧 56 栏 + 右主区的双栏中枢，概览页是「统计磁贴 + 最近回流时间线 + 去发现卡 + 钉住工具」；三辐 /info /community /tool 各自顶部带色点标题。视觉是单色「墨灰 Ink」+ 三辐色（资讯蓝 / 社区绿 / 工具紫）仅用于小圆点。问题：导航与品牌隐喻（中枢↔辐射、回流）几乎没有被视觉表达出来，信息密度与「极客感」也不足。
第一部分

竞品速览：他们的布局与可借鉴点

按「与 ideall 的关系」选取四类参照：阅读/聚合（信息回流）、本地优先知识工具（数据自持）、命令优先/壳层（极客效率）、视觉趋势（取势）。每张卡只留一句布局特征 + 一个可迁移点。

Readwise Reader

read-it-later · 三栏阅读

左侧固定视图（Home / Library / Feed / Search）+ 可拖拽自定义视图；正文「长文阅读模式」收起所有侧栏。核心是 Library（高信噪·手动收藏）vs Feed（低信噪·订阅推送）的二分。

借鉴：把 ideall 的「书签=高信噪」与「订阅=信息流」做成显式二分；阅读时一键进专注模式收起侧栏。

Folo（开源 AI RSS）

RSSNext/Folo · 三列

与 ideall 最像的开源对照：订阅列表 ｜ 条目列表 ｜ 正文三列，顶部按内容类型切视图（文章/社交/图片/视频/音频）；AI 翻译、摘要、每日 AI 摘要。

借鉴：多内容类型分视图 + 时间线 AI 摘要，正是 ideall 「资讯」可直接对标的形态。

Notion（2026 新侧栏）

tabbed sidebar

侧栏顶部改为标签页（Home / AI Chats / Meeting / Inbox），降低单一长列表密度，分区可自定义开关，移动端「点几下就到」。

借鉴：用「分标签的侧栏」收纳 home 子页 + AI + 收件箱，替代现在的横向 header。

Raycast / Arc

command-first · 竖向侧栏

命令栏一个输入框搞定「找标签页/历史/书签/执行动作」，键盘不离手；竖向侧栏吃满屏高，支持 Spaces、置顶、可折叠。

借鉴：把 ⌘K 从配角升为主角（导航的脊柱）；竖向可折叠图标轨替代横向 header。

Obsidian / Logseq

local-first · 面板 + 图谱

文件树左 / 编辑中 / 面板右，多窗格并排；本地 Markdown、数据自持；图谱视图是其标志性视觉（力导向、颜色分组、过滤）。

借鉴：「数据落在本机」的安全感要在 UI 里被看见；图谱可视化天然适配中枢辐射隐喻。

Heptabase

spatial · 白板画布

论点：理解信息需要空间化组织。核心单元是 Card，核心画布是无限 2D Whiteboard，卡片可空间排布、连线、嵌套白板；左栏像浏览器标签。

借鉴：为「中枢星系」方向提供范式——home 可做成可缩放的空间画布而非列表。

Raindrop.io / Feedly

多视图 · 聚合

Raindrop 支持 List / Grid / Masonry / Headlines 多视图，把 RSS、书签、PDF、视频聚到一个可搜索系统；Feedly 列表清爽、明亮配色、可去杂。

借鉴：同一份订阅流给多种密度视图切换；「聚合到一处可搜索」正是 ideall 的中枢主张。

Linear

dark-first · 仪表盘密度

近黑画布 + 极细字 + 单一克制强调色（每屏只给主操作）；冷灰单色阶、Inter Variable、用亮度层级而非粗细层级；侧栏调暗让内容区领先。

借鉴：「墨阶指挥所」方向的直接参照——把现有墨灰推到 dark-first 专业工具质感。

腾讯 ima

AI 工作台 · 中文受众

启动即见中央搜索框，左栏笔记/知识库/历史，框下「文档解读 / 智能写作」；模块化整合搜索+写作+笔记，拖拽导入 PDF/Word/MD。

借鉴：中文专业受众已习惯「中央 AI/搜索框」作为首屏主入口——可作 home 的英雄区。
视觉大势（2026）。 Bento 便当网格已成主流：ProductHunt 前 100 SaaS 约 67% 采用模块化网格，磁贴大小=信息重要度（非数据量），软圆角 + 每格微底色/阴影；据称比线性布局扫读更快。→ ideall 的概览页正适合从「五个等大磁贴」升级为真正的便当看板。
第二部分

提炼：迁移到 ideall 的 7 条原则

把上面的模式套回 ideall 的定位（开源·本地优先·供应商中立的个人信息工作台；受众=信息密集型专业人士 + 数据自持极客；隐喻=中枢↔辐射↔回流）后，得到贯穿四套方案的共同主线。

01

让「中枢辐射 + 回流」被看见

现在隐喻只活在文案里。新方案要用导航结构、动效或空间关系把「发现→带回中枢」这条主路径显性化——这是 ideall 区别于普通 RSS/书签器的根。

02

命令台升为导航脊柱

对标 Raycast/Arc：⌘K 不只是搜索，而是跳转 + 执行 + 订阅 + 回流的统一入口，匹配极客受众「键盘不离手」。每套方案都强化它的视觉存在感。

03

信噪二分：书签（高信噪）/ 订阅（流）

借 Readwise 的 Library/Feed 思路，让「我主动收藏的」和「自动推来的」在布局上分明，降低信息焦虑。

04

同一数据，多密度视图

借 Raindrop/Feedly：订阅流与列表支持列表 / 卡片 / 杂志 / 紧凑多视图切换，兼顾「快速扫读」与「沉浸阅读」两种作业。

05

本地优先要有安全感的视觉锚点

「数据落在本机 / 端到端同步 / BYO-key」是卖点，应有稳定可见的状态锚（本机芯片、存储条、同步态），而非藏在菜单里。

06

密度与克制并存

借 Linear：信息密但不吵——单色阶 + 单一强调色（每屏一个主操作），用亮度/间距分层，而非到处加粗加色。三辐色作为语义点缀而非填充。

07

桌面与移动同构降级

Tauri 桌面 + 移动同一套信息架构：桌面的「轨 + 栏 + 主 + 阅读面板」在移动端降级为「底部标签 + 单栏下钻 + 命令抽屉」，不另起炉灶。

方案 A

墨阶指挥所 · Ink Command

Linear · Raycast · Arc · Warp 谱系 — dark-first / 命令优先 / 仪表盘密度

把现有「墨灰 Ink」一路推到 dark-first 的专业工具质感：近黑画布、单一克制的酸性绿强调色、竖向可折叠图标轨 + 上下文栏 + 主区，⌘K 命令台是导航的脊柱。为「信息密集型 + 极客」受众做的高密度指挥中心。

方案 B

中枢星系 · Hub Galaxy

Obsidian 图谱 · Heptabase 空间画布 · Arc Spaces 谱系 — 把「中枢↔辐射↔回流」做成可见的空间

最大胆的一套：home 不是列表，而是一张可缩放的「星系画布」。中枢「我的」是发光内核，资讯/社区/工具是环绕轨道上的辐射节点，订阅与实体是更外圈的小星，连线即「回流」。三辐色升级为发光渐变系统，做出「别家没有」的辨识度。

方案 C

阅读编辑部 · Reading Room

Readwise Reader · Feedly · Folo 谱系 — 内容优先 / 暖纸色 / 三栏阅读 / 衬线标题

  • 三栏 + 信噪二分是成熟范式，学习成本低、稳
  • 暖色编辑感温度高，长时间使用不疲劳
△ 取舍
  • 「工具/极客感」相对弱，偏内容消费而非操作
  • 与 Readwise/Folo 形态接近，差异化要靠回流与本地优先
  • 暖纸色对部分重度暗色用户不友好（需夜读模式）
最适合：把 ideall 的核心价值押在「读得更好 / 信息消费」、希望产品有温度与可读性的你。体验最顺手、对泛用户最友好。
方案 D

便当工作台 · Bento Workbench

Notion 2026 分标签侧栏 · Bento 网格大势 · 腾讯 ima 中央 AI 框 谱系 — 柔和 / 模块化 / 可配置

怎么选：按你的首要目标对号入座

想最稳地立刻提升「专业感」

→ 选 A 墨阶指挥所。与现有墨灰一脉相承，迁移成本最低，立竿见影。

想做出「记得住、传得开」的差异化

→ 选 B 中枢星系。把品牌隐喻变成画面，但要接受更高工程量。

把核心价值押在「读得更好」

→ 选 C 阅读编辑部。三栏阅读 + 今日简报，体验最顺手。

要亲和、可定制、受众更宽

→ 选 D 便当工作台。现代 SaaS 质感，最安全的演进。

我的推荐：A 为底 + B 作 signature（可分两步走）

结合 ideall 的定位（本地优先、供应商中立、面向信息密集型专业人士与数据自持极客）与战略（要差异化护城河、零后端即完整）：

第一步，以 A · 墨阶指挥所 作为日常工作骨架——风险最低、最贴受众、把现有墨灰审美升级到位；

第二步，把 B · 中枢星系 作为 home 的「signature 首屏（星系态）」与 A 的列表态并存，做出独此一家的辨识度；资讯页可吸收 C 的三栏阅读，概览页可吸收 D 的便当可配置。

如果只想一步到位选一套：偏「工具/极客」选 A，偏「破圈/品牌」选 B，偏「阅读」选 C，偏「亲和/泛用」选 D。这是设计判断，最终由你定。

选定之后我会怎么落地

告诉我选哪套（或哪种组合），我就按 ideall 现有技术栈实现：React 19 + Tailwind v4 + shadcn，复用 @protocol 端口与 IndexedDB 数据层，遵循「默认 Server Component、仅交互组件 use client」「文案中文」「protocol 纯度」等既有约定。落地顺序建议：① 全局壳层与导航 → ② home 概览 → ③ 订阅/书签/资源/发布/AI 子页 → ④ info/community/tool 三辐。可每套方向开独立分支并行对比。
